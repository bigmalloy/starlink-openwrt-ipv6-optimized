OPENWRT SCRIPT COMPARISON & CHANGES (starlink-setup.sh)
--------------------------------------------------------
Repo: github.com/bigmalloy/starlink-openwrt
Target: OpenWrt 25.12.0, GL-iNet Beryl AX (MT3000)

CHANGE 1 — ip6assign: 60 -> 64
  Old: uci set network.lan.ip6assign='60'
  New: uci set network.lan.ip6assign='64'
  Reason: Official Starlink router does /56 -> /64 directly. The /60
          intermediary is unnecessary for a single-LAN setup and doesn't
          match observed behaviour.

CHANGE 2 — odhcpd prefix lifetime options (wrong UCI names)
  Old: uci set dhcp.lan.preferred_lft='3600'
       uci set dhcp.lan.valid_lft='7200'
  New: uci set dhcp.lan.max_preferred_lifetime='3600'
       uci set dhcp.lan.max_valid_lifetime='7200'
  Reason: preferred_lft and valid_lft are NOT valid odhcpd UCI options.
          They were silently ignored, meaning the Starlink short-lifetime
          fix was never actually applied. odhcpd was re-advertising
          Starlink's short lifetimes verbatim to LAN clients, causing
          prefix churn. The correct names per the odhcpd README are
          max_preferred_lifetime and max_valid_lifetime.

CHANGE 3 — verification section updated to match new option names

Settings confirmed correct (no change needed):
  - proto='dhcpv6', reqaddress='try', reqprefix='auto'    DHCPv6-PD fine
  - peerdns='0' + explicit Cloudflare/Google DNS          correct override
  - ra='server', dhcpv6='server'                          SLAAC + stateful fallback
  - ra_default='1'                                        advertise default route always
  - ra_maxinterval='60', ra_mininterval='30'              fast prefix propagation on renumber
  - ra_lifetime='600'                                     >= 3 * ra_maxinterval (RFC 9096)
  - accept_ra=2 sysctl                                    required — Linux ignores RA when
                                                          forwarding=1 unless accept_ra=2
  - ULA prefix                                            auto-generated by OpenWrt on
                                                          first boot, no explicit config needed


KEY DIFFERENCES: OFFICIAL ROUTER vs OPENWRT SCRIPT
---------------------------------------------------
1. Official router manages prefix changes internally (it IS the Starlink
   firmware). The OpenWrt script needs ra_maxinterval=60 + max_preferred/
   valid_lifetime to compensate for Starlink's short PD lifetimes upstream.

2. Official router: /56 -> /64 direct.
   OpenWrt script (fixed): /56 -> /64 direct (was /56 -> /60 -> /64).

3. Official router uses transition protocols (6rd, DS-Lite, MAP-E/T) as
   installed packages. Starlink residential does not use these in practice.

4. thc-ipv6 in SpaceX firmware is an internal security/testing toolkit,
   not relevant to production config.


ODHCPD REFERENCE (from github.com/openwrt/odhcpd README, 2026)
----------------------------------------------------------------
Key LAN (dhcp section) options for Starlink use:

  ra                    server|relay|hybrid|disabled
  dhcpv6                server|relay|hybrid|disabled
  ra_default            0=default, 1=ignore no public addr, 2=ignore all
  ra_slaac              1 (default) — advertise SLAAC prefix (A flag in PIO)
  ra_flags              other-config (default) — M=0 O=1, SLAAC + stateless DHCPv6
  ra_maxinterval        600 (default), set to 60 for fast Starlink renumbering
  ra_mininterval        200 (default), set to 30 for fast Starlink renumbering
  ra_lifetime           2700 (default), set to 600 (>= 3 * ra_maxinterval)
  max_preferred_lifetime 45m (default) — cap on preferred lifetime advertised to LAN
  max_valid_lifetime    90m (default)  — cap on valid lifetime advertised to LAN

Key WAN (network.wan6) options:
  proto                 dhcpv6
  reqaddress            try|force|none
  reqprefix             auto|no|0-64
  peerdns               0 (disable Starlink DNS)
